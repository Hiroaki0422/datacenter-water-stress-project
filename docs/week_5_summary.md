## v1.0 results in detail

### Validation scores

| Metric | v0 baseline | v1 (5-fold mean) | v1 (LOO worst) |
|---|---:|---:|---:|
| RMSE | 0.755 L/kWh | **0.276** | 0.869 (Meta) |
| MAE | 0.661 L/kWh | **0.197** | 0.816 (Meta) |
| R² | -1.149 | **0.651** | -717.0 (Meta) |

**Verdict:** v1 beats v0 by 63% on RMSE in the 5-fold test. The LOO test exposes a real generalization failure for Meta (R² = -717 because Meta's 16 air-cooled rows are the only examples of `cooling_type='air'` in the training set).

### Feature importance

| Feature | Gain |
|---|---:|
| `cooling_type_evaporative` | 0.466 |
| `cooling_type_air` | 0.381 |
| `latitude` | 0.040 |
| `longitude` | 0.033 |
| `operator_AWS` | 0.022 |
| `operator_Google` | 0.020 |
| `operator_Meta` | 0.014 |
| `is_aggregate_True` | 0.008 |
| `report_year` | 0.006 |
| `operator_Microsoft` | 0.005 |
| `is_aggregate_False` | 0.004 |

**~85% of the model's gain comes from the binary air-vs-evaporative split.** The model is essentially a 1.5-feature model (cooling type, with a small operator/geo adjustment). This is the **right physical axis** (air cooling is 5-10× more water-efficient than evaporative), but the binary nature means the model has only 16 air-cooled rows (all Meta) to learn from.

## What v1.0 means for journalism

The single most important headline change:

> **v0 said: US data centers use 237 B L/year of water.**
> **v1 (ML-corrected) says: US data centers use 144 B L/year of water.**
> **The difference (39%) is mostly the v0 physics formula's cooling-penalty double-counting**, not a real-world reduction in water use. The actual water use is probably somewhere between v0 and v1, closer to v1.

Both numbers should be reported, with the v0 formula's known limitations cited (methodology.md Section 9). v1's 144 B L/year is the **best current estimate**; v0's 237 B L/year is the **conservative upper bound** (assumes every facility runs evaporative cooling at full load).
